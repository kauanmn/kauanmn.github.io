O que é aprendizado?

Aprendizado é a capacidade de se adaptar, modificar e melhorar seu comportamento e suas respostas, sendo, portanto, uma das propriedades mais importantes dos seres humanos inteligentes, sejam eles humanos ou não

Raciocínio indutivo

Dada uma linguagem de descrição usada para expressar possíveis hipóteses, um conhecimento preliminar, um conjunto de exemplos positivos, e um conjunto de exemplos negativos, o Aprendizado Indutivo pode ser entendido como o problema de se encontrar uma hipótese que cubra todos os exemplos positivos e não cubra nenhum dos exemplos negativos.

Conceito de Machine Learning (ML)

Machine Learning (ML) é um subcampo da inteligência artificial (IA) que se concentra no desenvolvimento de algoritmos e modelos que permitem que sistemas computacionais aprendam e melhorem com a experiência.

O objetivo do ML é permitir que um sistema aprenda a partir de dados em vez de ser programado explicitamente para realizar uma tarefa.

Isso é feito por meio do uso de algoritmos que analisam dados, identificam padrões e usam esses padrões para fazer previsões ou tomar decisões.

Algoritmo

Um algoritmo é um conjunto de atividades que podem ser executadas passo a passo para resolver problemas.

  • Um programa de computador é uma das formas de representar um algoritmo
  • exemplo: crie um algoritmo para exibir o resultado da multiplicação de dois números. Passo a passo (atividades):
    • passo 1: receber os dois números que são multiplicados
    • passo 2: multiplicar os números
    • passo 3: mostrar o resultado obtido da multiplicação

Programação Tradicional x Machine Learning

Programação Tradicional

Regras + Dados → Algoritmo → Saída

Exemplo: algoritmo em Python para exibir o resultado da multiplicação de dois números:

num1 = float(input('Insira o primeiro número: '))
num2 = float(input('Insira o segundo número: '))

print(f"O resultado da multiplicação é {num1 * num2}")

　

Machine Learning

Dados → Algoritmo de ML (pré-existente) → Modelo (Regras) Mais dados → Modelo → Saídas

　

Passo a passo do ML:

  1. Coleta de dados
  2. Exploração e Preparação
  3. Treinamento do Modelo
  4. Avaliação do Modelo
  5. Otimização do Modelo (volta para o passo 1)

Problemas para algoritmos de IA

Os principais tipos de problemas são:

  • previsão de valores
  • classificação em categorias
  • associação entre itens
  • agrupamentos automáticos
  • detecção de anormalidades
  • busca de solução otimizada

　

Exemplos:

  • dadas certas características de um filme/série, qual é a previsão de arrecadação do filme? (previsão de valor)

  • o roteiro X é considerado bom roteiro de acordo com os padrões da produtora? (classificação)

  • dado o histórico de compras de um usuário em uma plataforma de streaming, quais filmes ou séries são mais propensos a serem recomendados para ele? (associação entre itens)

Tipos de Machine Learning

  • Aprendizagem Supervisionada
  • Aprendizagem Não-Supervisionada
  • Aprendizagem Semi-Supervisionada
  • Aprendizagem por Reforço

Aprendizagem supervisionada: classificação

A classificação preocupa-se com a construção de modelos que separam os dados em classes distintas.

Aprendizagem supervisionada: regressão

  • Uma variável independente x, explica a variação em outra variável, chamada de variável dependente y. Este relacionamento existe em apenas uma direção: variáveis independentes → variáveis dependentes

  • Análise de regressão: metodologia estatística que usa a relação entre duas ou mais variáveis quantitativas de tal forma que uma variável possa ser predita a partir de outra

  • A regressão preocupa-se com a construção de modelos que possam prever valores

SVM (Support Vector Machine)

  • pode ser usado para classificação ou regressão
  • o SVM padrão toma como entrada um conjunto de dados e prediz, para cada entrada dada, qual de duas possíveis classes a entrada faz parte
    • isso faz do SVM um classificador linear binário não probabilístico
  • em caso de outlier, a SVM busca a melhor forma possível de classificação e, se necessário, desconsidera o outlier
  • funciona bem em domínios complicados em que existe uma clara margem de separação
  • não funciona bem com conjuntos de dados muito grandes, pois exige inversão de matriz (alta complexidade computacional) -> O(n³)
  • não funciona bem em conjuntos de dados com muito ruído
  • se as classes estiverem muito sobrepostas, deve-se usar apenas evidências independentes (devido ao fato de não ser muito bom com dados com muitos ruídos)
