Conceitos Essenciais de Big Data

  • Dado: registro solto, aleatório, sem qualquer análise.
    • Um dado sozinho não diz nada, não tem muito valor
  • Informação: dado após o processo de interpretação
  • Conhecimento: informação que passa por processo de análise e trabalho

Dados → Informação → Conhecimento → Tomada de Decisão → Ação → Benefícios

Importante:

O principal objetivo é extrair informações dos dados e transformá-las em conhecimento que possa ser usado para a tomada de decisões.

Dados Estruturados x Não Estruturados

  • Dados Estruturados: dados que contêm uma organização para serem recuperados
    • como se fossem etiquetas, linhas e colunas que identificam diversos pontos sobre aquela informação e tornam o trabalho da tecnologia bem simplificado
    • imagine sua lista de compras no excel, onde existem colunas "Nome", "Qtd" e "Preço". Você sabe previamente qual informação estará em cada coluna de cada linha subsequente
  • Dados Não Estruturados: significa que eles não têm relação entre si e nenhuma estrutura definida
    • são, por exemplo, posts no facebook, vídeos, fotos, tweets, geolocalização, comportamento do usuário etc.
  • Dados Semiestruturados: forma de dados estruturados que não está em conformidade com a estrutura formal de modelos de dados associados a bancos de dados relacionais ou outras formas de tabelas de dados, mas mesmo assim contém tags ou outros marcadores para separar elementos semânticos e impor hierarquias de registros e campos nos dados
    • também conhecida como estrutura autodescritiva, ex.: JSON e XML

Metadados

  • do ponto de vista técnico, não é uma estrutura de dados separada, mas é um dos elementos mais importantes para a análise de big data e soluções de big data
  • fornece informações adicionais sobre um conjunto específico de dados
    • por exemplo, em um conjunto de fotografias, os metadados podem descrever quando e onde as fotos foram tiradas. Os metadados fornecem campos apara datas e locais que, por si só, podem ser considerados estruturados
    • por esse motivo, os metadados são usados por soluções de big data para análise inicial

Data Science

  • matemática e estatística
  • ciência da computação
  • conhecimento das áreas de negócio

Data-Driven Culture

  • empresa com cultura orientada a dados
  • reconhece a importância do big data e como a análise desses dados pode levar a empresa a criar produtos e serviços personalizados e totalmente aderentes às necessidades do público

Análise de Dados

Tipos de análises:

  1. Descritiva: o que aconteceu?
  2. Diagnóstica: por que isso aconteceu?
  3. Preditiva: o que acontecerá?
  4. Prescritiva: o que deve ser feito?

Ordem das análises: descritiva → diagnóstica → preditiva → prescritiva

Análise Descritiva

  • usa dados históricos e atuais de várias fontes para descrever o estado atual, identificando tendências e padrões
  • como a análise descritiva não emite julgamento de valor, ela é indicada para visualizar os dados e entender o impacto no presente, mas sem fazer relação com o passado ou o futuro
  • ajuda a tomar decisões imediatas com tranquilidade e segurança
  • pode incluir cálculos de média, mediana, moda, desvio padrão, gráficos de barras, histogramas, etc.

Análise Diagnóstica

  • usa dados (muitas vezes gerados por análises descritivas) para descobrir os fatores ou motivos do desempenho passado
  • particularmente útil quando há problemas ou fenômenos indesejados que precisam ser compreendidos em termos de suas causas

Análise Preditiva

  • aplica técnicas como modelagem estatística, previsão e aprendizado de máquina à saída de análises descritivas e de diagnóstico para fazer previsões sobre resultados futuros
  • inclui técnicas como regressão linear, regressão logística, árvores de decisão, redes neurais, etc.

Análise Prescritiva

  • análise avançada que envolve a aplicação de testes e outras técnicas para recomendar soluções específicas que fornecerão os resultados desejados
  • vai além da análise preditiva, fornecendo recomendações ou sugestões de ação com base em insights extraídos dos dados
  • pode incluir otimização, simulação e modelagem de cenários
