IoT (Internet das Coisas) continuação

TpM | Fase 2 - Requisitos

O levantamento de requisitos é uma parte fundamental para o sucesso do desenvolvimento da solução IoT.

  • O uso do modelo de referência segue o sentido top-down. Isto tem por objetivo partir do negócio, avaliando o que deve ser exibido para a tomada de decisões
  • O levantamento de requisitos são de três tipos: requisitos funcionais, não funcinoais e desejáveis

Tipos de Requisitos

De forma a tornar a tarefa mais simples e lógica, os requisitos podem ser classificados como funcionais, não funcionais e desejáveis

  • Os requisitos funcionais e não funcionais são essenciais para o funcionamento do sistema porque definem o que o sistema deve fazer e como ele deve fazer

  • Requisitos funcionais: requisitos que definem o que o sistema deve fazer (comportamentos, resultados, capacidades)

  • Requisitos não funcionais estão relacionados ao uso da aplicação em termos de desempenho, usabilidade, segurança, tecnologias usadas etc. Ou seja, como será entregue ao usuário.

  • Os requisitos desejáveis são opcionais. Eles podem ser especificados, mas não são necessários

  • Para listar os requisitos, espera-se que eles atendam a algumas características como ser descritos claramente, sem ambiguidades e consistente com todos os aspectos significativos do sistema proposto
