IoT (Internet das Coisas)

Para pensar

  • o que significa IoT e como é definido?
  • como IoT pode ser usado em uma empresa?
  • existe uma forma de desenvolver IoT?
  • quais as partes de IoT?
  • por que os projetos IoT falham?
  • por que existem tantas propostas de rede sem fio para IoT?

O que é IoT?

Descreve dispositivos com sensores, habilidade de processamento, software e outras tecnologias que conectam e trocam dados com outros dispositivos e sistems através da Internet ou outras redes de comunicação

Equívoco

  • Um equívoco frequente em IoT é dar mais relevância à tecnologia do que ao que de fato a solução vai acrescentar ao negócio do cliente

  • Esse é um erro recorrente porque, para o cliente, a tecnologia usada é irrelevante de forma geral, uma vez que o objetivo da solução para o negócio deve ser:

    • aumentar a eficiência dos processos
    • diminuir desperdício
    • economizar recursos
    • promover a sustentabilidade
    • segurança patrimonial etc

Importância dos dados brutos e processados

Os dados brutos são processados e geram conhecimento que, por sua vez, geram sabedoria para a tomada de decisão.

Dados -> Informação -> Conhecimento -> Sabedoria

Solução IoT e Tecnologia

  • No contato com o negócio, deve ficar muito bem definido onde a solução IoT criada vai se pagar
  • Observar que o ganho obtido com a solução IoT deve, necessariamente, ser viável economicamente
  • Os ganhos podem ser tangíveis ou intangíveis. Entretanto, o aspecto financeiro é fundamental
  • Se essa questão financeira não ficar bem definida desde o início das entrevistas com o cliente, a chance de insucesso é grande
  • Tecnologia deve atender às necessidades do negócio e não se deve ficar preso a tecnologias ao criar uma SOLUÇÃO IoT

Informações na Internet

  • As outras informações podem estar, inclusive, na Internet. Por exemplo, a previsão do preço de um ativo ou do tempo
  • No sentido mais simples, o conhecimento é a informação refinada e tratada, usando diferentes fontes de dados
  • A sabedoria nasce, portanto, do conhecimento decorrente da experiência
  • DEnquanto o conhecimento muda com o tempo, a sabedoria é atempora, e tudo começa com a aquisição de dados
  • No topo está a capacidade de decisão, que sempre vai caber ao negócio
  • A IoT mudou drasticamente e ampliou seu escopo de aplicações em poucos anos, atingindo todas as áreas da atividade humana, da biologia à tecnologia às ciências sociais

Por que projetos IoT falham?

  • Cultura organizacional
  • Falta de recursos
  • Falta de planejamento
  • Ignorância das tecnologias
  • Falta de profissionais qualificados
  • Custos ocultos
  • Curva de aprendizagem
  • Redes de sensores mal dimensionadas ou inadequadas
  • Muitas opções de rede sem fio

Rede de sensores sem fio vs. IoT

  • A rede de sensores sem fio é o elo fraco da IoT
  • Dependência do ambiente em que será instalado
  • Somente é possível avaliar efetivamente após a instalação da rede
  • Necessidade de gerência da rede de sensor sem fio

Modelo de Referência para IoT

O objetivo de um modelo de referência é fornecer definições e descrições claras que possam ser aplicadas com precisão a elementos e funções de sistemas e aplicativos IoT, permitindo:

  • Simplificar: ajuda a quebrar sistemas complexos para que cada parte seja mais compreensível
  • Esclarecer: fornece informações adicionais para identificar, com precisão, os níveis da IoT e estabelecer terminologia
  • Identificar: identifica onde tipos específicos de processamento são otimizados em diferentes partes do sistema
  • Padronizar: fornece uma primeira etapa para permitir que fornecedores criem produtos de IoT que funcionem entre si
  • Organizar: torna a IoT real e acessível

Modelo de Referência CISCO

7. Colaboração e processos
6. Aplicação
5. Abstração de dados
4. Acumulação de dados
3. Edge computing
2. Conectividade
1. Dispositivos físicos e controladores
  • O IoT Reference Model também permite que o processamento que ocorre em cada nível varie de trivial a complexo, dependendo da situação
  • O modelo descreve como as tarefas em cada nível devem ser tratadas para manter a simplicidade, permitem alta escalabilidade e garantem a capacidade de suporte
  • Finalmente, o modelo define as funções necessárias para um sistema IoT para ser completo
  • É importante notar que os dados fluem em ambas direções

Open Source Reference Model IoT

  • Embora o modelo de referência apresentado pela CISCO seja completo, não é de fácil assimilação para aplicações em pequenas e médias empresas
  • Então, é apresentada um modelo de referência mais acessível de 6 níveis

6 - Exibição: trata de como mostrar as informações para a tomada de decisão do negócio

5 - Abstração: define como usar os dados armazenadas para realizar a transformação de informação em conhecimento

4 - Armazenamento: define como será o armazenamento dos dados. Pode ser na nuvem, no cliente, pode usar blockchain etc.

3 - Borda: define como será o elemento que conecta a solução de IoT com a Internet. Este elemento de borda atua tanto na rede IoT, quanto na Internet, fazendo a ligação entre os dois mundos. A solução IoT converge para este elemento de borda.

2 - Conectividade: define como será a conectividade entre os dispositivos e o elemento de borda. Podem ser soluções sem fio, cabeadas, híbridas etc.

1 - Processamento Local (Sensor / Atuador): define os nós sensores/atuadores que existirão na solução

TpM (Three-Phase Methodology)

A Three-Phase Methodology (TpM) é uma abordagem estruturada para o desenvolvimento e implementação de soluções, dividindo o ciclo de desenvolvimento em três fases principais: negócio, requisitos e desenvolvimento.

  • Inicialmente, é necessário fazer a definição do que é uma solução IoT, usando o TpM como base
  • Usando uma análise lógica, é necessário avaliar as necessidades do negócio como primeira fase, no qual a solução vai beneficiar
  • A segunda fase é levantar os requisitos da solução, usando o modelo de referência de 6 níveis. Cada nível tem as suas especificidades que devem ser levantadas antes da identificação de tecnologias
  • Na terceira fase, também usando o modelo de referência, é realizada a implementação

Solução IoT

Uma solução IoT usa informações capturadas do mundo físico local e/ou virtual pela Internet, para atender aos objetivos de um negócio, com a identificação dos requisitos necessários para a implementação usando diferentes tecnologias.

Agentes e Funções

  • O primeiro ponto a destacar é a necessidade da identificação dos papeis envolvidos na criação da solução IoT, caracterizando os agentes e suas funções
  • Toda metodologia de desenvolvimento precisa ter seus agentes e papeis bem definidos
  • Esta definição é importante para a organização e estruturação do trabalho que deve ser feito em parceria entre os representantes do negócio e a equipe de desenvolvimento
  • Esta organização ajuda a integrar a equipe, pois as responsabilidades e necessidades são explicitadas e bem delineadas
  • Entretanto, é fundamental a hierarquia para estabelecimento do fluxo das informações para não existirem questões mal resolvidas

Participantes do desenvolvimento da solução IoT

  • Cliente
  • Gerente de Projeto
  • Gerente de Desenvolvimento
  • Equipe de desenvolvimento (multidisciplinar)

Fluxograma de desenvolvimento

Entender como estão encadeadas as fases é fundamental, mas principalmente quais são as saídas de cada fase, criando um conjunto de documentos que permita a criação de informações formais para a criação da solução IoT

Empreendimento e solução de fábrica

  • Uma solução IoT tem por objetivo capturar informações para serem analisadas e permitir que o empreendimento tome decisões baseadas nos dados
  • Resgatando a diferença entre empreendimento e processos produtivos
  • Solução IoT está ligada aos interesses do empreendimento e não nos processos
  • Exemplo da fábrica de sorvetes:
    • a parte da corporação que trata da parte estratégica, principalmente financeira
    • nível de fábrica que cuida dos processos, também chamado de chão de fábrica

Empreendimento e processos produtivos

  • A TpM atende ao empreendimento e não aos processos produtivos
  • A TpM é uma metodologia para desenvolver uma solução IoT para o empreendimento, coletando dados brutos para serem transformados em informação e conhecimento para tomada de decisão
  • A TpM naõ tem como propósito agir nos processos produtivos como, por exemplo, controlar máquinas na produção
  • Essa ação somente existiria como um decisão do empreendimento para atender a uma decisão estratégica

TpM | Fase 1 - Negócio

Entendendo o negócio

  • A figura mostra como as áreas que afetam o desenvolvimento da solução IoT
  • O objetivo das três fases é ligar o negócio às coisas, através da solução IoT, usando regras do negócio, o conhecimento de um especialista e identificando as coisas
  • O objetivo que o gerente de projeto conduza a aproximação com o cliente para o levantamento de informações sobre o negócio
  • São realizadas entrevistas com os clientes com questões previamente preparadas
  • As informações obtidas vão balizar o desenvolvimento da solução IoT

Negócio

O negócio é o domínio de aplicação da solução de IoT e trata do objetivo da solução, o motivo pelo qual ela será criada

  • É o norteador do desenvolvimento, onde são definidas as prioridades, condições e informações que são importantes e geradas pela solução de IoT
  • Nessa etapa, o gerente de projeto é apresentado ao problema que deve ser resolvido, ou seja, ao negócio
  • Ele levanta questões sobre o problema e deve entender o que se deseja da solução
  • O negócio varia de projeto para projeto, mesmo quando se tratar de projetos na mesma área de atuação porque cada empresa tem suas peculiaridades
  • Identifica a necessidade de blockchain quando da existência de partes interessadas que devem compartilhar informação

Regras de Negócio

As regras de negócio são premissas e restrições aplicadas a uma operação comercial de uma empresa que precisam ser atendidas para que o negócio funcione para retorno do investimento.

Especialista

Há a necessidade de um especialista nas coisas do negócio para que as informações colhidas sejam suficientes para dar subsídio às demais fases. Ex.: engenheiro de software

Coisas

As coisas são as entidades que devem ser monitoradas ou manipuladas.

  • é importante entender que as coisas não são objetos, mas as grandezas que devem ser monitoradas/controladas
  • podem ser grandezas físicas como temperatura, ou controle de componentes como relés e câmeras
